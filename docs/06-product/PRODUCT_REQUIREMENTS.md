**产品名称**：Unveil — 揭棋对弈系统  
**版本**：v1.0 · 2026-06-18  
**定位**：面向课程验收的揭棋网络对弈与 AI 博弈平台

---

## 1. 产品定位

**一句话**：面向课程验收的揭棋对弈系统，重点展示面向对象设计、网络通信、规则校验、AI 博弈、棋谱复盘和工程化自检。

**优先级排序**（产品决策原则）：

```
规则稳定 > AI 可解释 > 复盘可追溯 > 一键运行 > 演示流程清晰 > 花哨 GUI
```

**不追求**：商业级棋牌 App 体验、用户增长、付费体系。  
**追求**：老师 3 分钟看懂、8 分钟演示不翻车、同学可按公共接口互联。

---

## 2. 用户角色与场景

| 角色 | 典型场景 | 使用入口 |
|------|----------|----------|
| **普通玩家** | 课后与同学联网下一盘揭棋；研究暗子翻子策略 | `client-ws` 控制台客户端 |
| **AI 挑战者** | 单人练习，选择 Easy/Medium/Hard 人机对战 | 交互菜单「人机对战」或 `ai-ws` |
| **教师验收者** | 检查协议、规则、AI、文档是否达标；现场观看演示 | `verify.ps1` + `DEMO_SCRIPT.md` + 双客户端 |
| **观战者**（扩展） | 旁观他人对局学习（非主路径） | 协议预留，当前为实验性 |

---

## 3. 用户痛点

| 痛点 | 现状（无系统时） | Unveil 解决方案 |
|------|------------------|-----------------|
| **规则判断困难** | 揭棋暗子/翻子/强化士象规则复杂，人工易错 | 服务端 `RuleValidator` 权威校验，双端一致拒绝 |
| **网络状态不同步** | 自行 socket 易出现棋盘不一致 | WebSocket JSON 广播 `moveResult` + 服务器棋盘快照 |
| **AI 作弊疑虑** | 简单 AI 可能「透视」暗子 | `createAiPublicView` + `AiFairnessTest` 约束 |
| **无法复盘** | 棋谱重放因翻子随机无法还原 | `ReplayTimeline` 逐步棋盘快照 |
| **验收/demo 不可复现** | 环境杂乱、无自检 | Maven 多模块 + `verify.ps1` + 演示脚本 |
| **组间无法互联** | 各组协议不一致 | `INTERFACE.typ` v3.0 权威协议 + PDF |

## 4. 核心功能（用户视角）

### 4.1 对弈

- 登录服务器，输入 `userId` 与密码
- 匹配对手或指定 AI 对战
- 控制台棋盘显示（红方在下、黑方在上）
- 输入坐标走子（如 `move e6 e5`）
- 实时看到对方走子、翻子结果、吃子
- 聊天、提和、认输、请求加时

### 4.2 规则保障

- 非法走子立即提示，棋盘不变
- 超时自动判负（65 秒）
- 终局自动判定：将死、困毙、和棋、长将长捉等
- 终局弹窗式摘要：胜者、原因、总步数

### 4.3 AI 对手

- 三档难度：入门（Easy）、进阶（Medium）、挑战（Hard）
- AI 思考有进度感（控制台输出思考时间）
- 不透视对手暗子，符合揭棋信息差
- 可在同一服务器与人类混排

### 4.4 棋谱与复盘

- 每局自动生成文字棋谱（`records/*.jieqi`）
- 终局后可「上一步 / 下一步」复盘
- 复盘显示当时棋盘（终局后可上帝视角）
- 支持导出路径提示，便于实验报告引用

### 4.5 工程体验

- 一个 Fat JAR 或 `mvn exec:java` 启动菜单
- 9 项菜单：服务器、客户端、AI、自检等
- Docker Compose 可选一键起服务

## 5. 产品闭环

```
启动程序（jieqi-app 菜单 / Fat JAR）
        ↓
   选择模式
   ├── WebSocket 真人对战
   ├── 人机对战（三档）
   └── AI 自动对弈（AI vs AI / AI 接入房间）
        ↓
   登录 → 匹配 → 准备 → 争先手 → 开局
        ↓
   对弈中：走子 / 聊天 / 提和 / 认输 / 加时
        ↓
   终局（将死 / 超时 / 认输 / 和棋 …）
        ↓
   查看摘要（胜者 / 原因 / 步数 / 棋谱路径）
        ↓
   复盘（逐步回看 ReplayTimeline）
        ↓
   导出棋谱 → 再来一局 / 返回大厅
```

**闭环验收标准**：任意一条路径从启动到终局摘要 ≤ 8 分钟可演示完毕，且无人工改棋盘。

## 7. 成功指标（可度量）

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 单元测试通过率 | 100% | `mvn test` |
| 验收项 C1–C20 通过 | ≥ 18/20 | `ACCEPTANCE_CRITERIA.md` 勾选 |
| 演示脚本时长 | 6–8 分钟 | `DEMO_SCRIPT.md` 彩排计时 |
| AI Easy 步时 | < 500ms | `PerformanceTest` |
| AI Medium/Hard 步时 | < 60s（含裕量） | 实战日志 |
| 非法走子拒绝率 | 100% | 非法走法测试集 |
| 协议字段一致性 | 与 INTERFACE.typ 零矛盾 | 人工 + 互操作测试 |

## 8. 非目标（明确不做）

- 商业运营、支付、账号体系
- 移动端 App
- 实时语音
- 反作弊 beyond 服务端校验
- 图形化 3D 棋盘

## 9. 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| 答辩现场环境缺 JDK 21 | 无法演示 | 提前打包 Fat JAR；U 盘备份 |
| 长将长捉边界争议 | 老师质疑规则 | 备注已知限制；引用 INTERFACE Q 清单 |
| AI 超时 | 演示卡顿 | 降级 Easy；调大 `timeLimitMs` |
| 端口占用 | 服务器起不来 | `TROUBLESHOOTING.md` 预案 |
