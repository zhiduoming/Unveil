## 1. 竞品选取

| 竞品类型 | 代表 | 选取理由 |
|----------|------|----------|
| **普通中国象棋 App** | 天天象棋、象棋巫师等 | 用户熟悉，但通常不支持揭棋 |
| **在线棋类平台** | ICC、lichess（象棋区）、各棋牌大厅 | 网络对弈成熟，规则引擎强，但非课设接口 |
| **简单 Java 棋类课设** | 典型单模块 Socket + 控制台课设 | 同课程对标物，工程与规则深度有限 |
| **本项目 Unveil** | 本仓库 | 课程定制，揭棋 + 互操作 + AI + 工程化 |

## 3. 差异化分析

1. **课程场景适配**：Unveil 按 BUPT 2026 公共 WebSocket 接口设计，端口、消息类型、坐标系与互操作文档可直接用于组间联调；商业 App 与课设评分标准不对齐。

2. **揭棋专用规则深度**：相比普通象棋 App 和简单课设，Unveil 实现暗子 virtualType、服务器随机翻子、强化士象、长将长捉等揭棋特有规则，并有对应单元测试与验收项 C1–C12。

3. **工程可验收性**：Maven 五模块、自检脚本、分层文档与演示脚本构成「可复现验收包」；在线平台功能更强但不提供面向课程评分点的可交付物结构。

## 4. SWOT 简表（本项目）

| | 有利 | 不利 |
|---|------|------|
| **内部** | 规则/AI/协议文档齐全；测试覆盖核心路径 | 无 GUI；长捉边界待强化 |
| **外部** | 课程明确要求揭棋+网络+AI；互操作加分 | 其他组可能用更炫前端 |

---

## 5. 定位结论

| 场景 | 推荐选择 |
|------|----------|
| 日常娱乐下象棋 | 商业 App |
| 引擎级传统象棋训练 | 在线平台 |
| **课程揭棋验收与互操作** | **Unveil** |
| 快速 demo 无揭棋 | 简单课设 |

Unveil 的目标不是在用户体验上超越商业棋牌产品，而是在**课程评分维度**（OO 设计、协议、规则正确性、AI 算法、测试与文档）上提供完整、可审计、可演示的解决方案。
