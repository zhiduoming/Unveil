## 2. 分步旅程表

| 步骤 | 用户操作 | 系统行为 | 界面/输出预期 | 可选分支 |
|------|----------|----------|---------------|----------|
| **S1 进入系统** | 运行 `mvn exec:java -f jieqi-app/pom.xml -am` 或 `java -jar unveil-jieqi.jar` | 显示 1–9 交互菜单 | 菜单含「启动 WS 服务器」「WS 客户端」等 | 直接命令行传参跳过菜单 |
| **S2 选择模式** | 输入 `3`（WS 客户端）或 `5`（人机） | 提示输入 URL / userId / 密码 / AI 等级 | 参数默认值 `ws://127.0.0.1:8887` | 选 TCP 客户端（附录 B） |
| **S3 登录** | 连接后输入 `login`（或自动登录） | 发送 `login` JSON；返回 `loginResponse` | `success: true` | 密码错误 → 提示失败 |
| **S4 匹配** | 输入 `match` | 服务器撮合两名等待玩家；返回 `matchSuccess` | 显示 `roomId`、`yourColor`、对手 ID | 人机模式：匹配 AI 房间 |
| **S5 准备** | 双方输入 `ready` | 双方就绪后进入争先手 | 提示等待对手 ready | — |
| **S6 争先手** | 输入 `first true` 或 `first false` | 服务器决定先手；广播 `gameStart` | 显示初始棋盘（仅将帅明） | — |
| **S7 开局** | 查看棋盘 | 服务器下发 `initialBoard`；开始计时 | ASCII 棋盘；当前行棋方高亮 | — |
| **S8 走子** | 输入 `move e6 e5`（示例） | 校验 → `moveResult`（含 flipResult）→ 切换回合 | 棋盘更新；若翻子显示真实 type | 非法走法 → `error` 不变盘 |
| **S9 对局中交互** | `chat 你好` / `draw` / `resign` | 广播聊天；处理和棋请求；认输终局 | 聊天显示在控制台 | 对方拒绝和棋 → 继续 |
| **S10 超时** | （无操作 65s） | 服务器判负 | `gameOver` reason=TIMEOUT | 可申请加时（若协议支持） |
| **S11 终局** | — | 广播 `gameOver`（winner + reason） | 显示 CHECKMATE / STALEMATE / DRAW 等 | — |
| **S12 查看结果** | 阅读终局摘要 | 客户端打印 `GameSummary` | 胜者、步数、棋谱路径 `records/xxx.jieqi` | — |
| **S13 复盘** | 输入 `replay` 或 `next` / `prev` | 发送 `replayRequest`；返回 `replayFrame` | 按帧显示历史棋盘与着法 | 终局后上帝视角可见全部暗子 |
| **S14 再来一局** | 输入 `rematch` | `rematchRequest` / 新 `gameStart` | 新房间或同房间重置 | 拒绝则返回大厅 |
| **S15 退出** | `quit` 或关闭终端 | 断开 WebSocket；房间清理 | 连接关闭提示 | 断线可能判负（实验性） |

## 3. 人机对战旅程差异

| 步骤 | 与真人对战差异 |
|------|----------------|
| S4 匹配 | 服务器分配 AI 对手（`ai_bot_*`），无需第二名玩家 |
| S8 走子 | 用户走子后 AI 自动思考 0.5–5s 并走子 |
| S9 | 无聊天心理战需求；可认输结束 |
| S13 复盘 | 与真人对战相同 |
